Automating AWS Infrastructure with Terraform

This guide delves into the power of Terraform for managing your Amazon Web Services (AWS) infrastructure. Terraform, an Infrastructure as Code (IaC) tool, allows you to define your cloud resources in human-readable configuration files, enabling consistent, repeatable, and version-controlled deployments. Say goodbye to manual clicks and hello to automated, scalable infrastructure!

We'll cover the essentials: setting up your environment, provisioning virtual machines (EC2), managing object storage (S3), and building custom virtual networks (VPC). Each topic is accompanied by practical Terraform code examples and clear instructions, along with placeholders for visual aids.

Getting Started: Prerequisites

  • An active AWS Account with appropriate IAM permissions.
  • AWS Command Line Interface (CLI) installed and configured.
  • Terraform CLI installed on your local development machine.
  • Familiarity with basic cloud computing and networking concepts.

1. Deploying EC2 Instances with Terraform

Amazon EC2 (Elastic Compute Cloud) provides scalable computing capacity in the AWS cloud. Terraform simplifies the process of defining and launching EC2 instances, ensuring your virtual servers are provisioned exactly as you intend.

Define Provider & Variables

Start by declaring the AWS provider and defining key variables like region, instance type, and AMI ID. 


# providers.tf
provider "aws" {
  region = var.aws_region
}

# variables.tf
variable "aws_region" {
  description = "The AWS region to deploy resources in"
  type        = string
  default     = "us-east-1"
}

variable "instance_type" {
  description = "EC2 instance type (e.g., t2.micro, m5.large)"
  type        = string
  default     = "t2.micro"
}

variable "ami_id" {
  description = "AMI ID for the EC2 instance (e.g., Amazon Linux 2 AMI for us-east-1)"
  type        = string
  default     = "ami-053b0d53c279acc90" # Example: Amazon Linux 2 AMI
}

Create Security Group & EC2 Resource

A security group acts as a virtual firewall. We'll create one to allow SSH and HTTP, then define the EC2 instance. 


# ec2.tf
resource "aws_security_group" "web_access_sg" {
  name        = "web-access-security-group"
  description = "Allow web and SSH access"
  vpc_id      = aws_vpc.main_vpc.id # Assuming VPC is defined elsewhere

  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"] # For demonstration, restrict in production
    description = "Allow SSH from anywhere"
  }

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"] # For demonstration, restrict in production
    description = "Allow HTTP from anywhere"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1" # Allow all outbound traffic
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Name = "WebAccessSG"
  }
}

resource "aws_instance" "my_web_server" {
  ami                    = var.ami_id
  instance_type          = var.instance_type
  key_name               = "your-ssh-key-name" # IMPORTANT: Replace with your actual key pair name
  vpc_security_group_ids = [aws_security_group.web_access_sg.id]
  # subnet_id              = aws_subnet.public_subnet.id # Uncomment if using custom VPC/subnet

  tags = {
    Name = "TerraformWebServer"
  }
}

output "web_server_public_ip" {
  description = "Public IP address of the web server"
  value       = aws_instance.my_web_server.public_ip
}

2. Managing S3 Buckets with Terraform

Amazon S3 (Simple Storage Service) offers highly scalable, durable, and available object storage. Terraform allows you to define S3 buckets, their configurations, and access policies with ease.

Create an S3 Bucket

Define a unique bucket name and specify its access control list (ACL). For static websites, you might set `acl = "public-read"`. 


# s3.tf
resource "aws_s3_bucket" "static_website_bucket" {
  bucket = "your-unique-static-website-name-2025" # IMPORTANT: Must be globally unique
  acl    = "private" # Start private, adjust for public website hosting

  tags = {
    Name        = "MyTerraformStaticWebsite"
    Environment = "Development"
  }
}

# Optional: Configure for static website hosting
resource "aws_s3_bucket_website_configuration" "website_config" {
  bucket = aws_s3_bucket.static_website_bucket.id

  index_document {
    suffix = "index.html"
  }

  error_document {
    key = "error.html"
  }
}

# Optional: Block all public access (recommended for most buckets)
resource "aws_s3_bucket_public_access_block" "block_public_access" {
  bucket = aws_s3_bucket.static_website_bucket.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

output "s3_bucket_endpoint" {
  description = "Website endpoint for the S3 bucket"
  value       = aws_s3_bucket_website_configuration.website_config.website_endpoint
}

Terraform Commands for S3

After defining your S3 resources, run these commands to apply the changes. 


# Initialize Terraform in your project directory
terraform init

# Review the plan before applying changes
terraform plan

# Apply the changes to create the S3 bucket
terraform apply --auto-approve

# To destroy the resources later
# terraform destroy --auto-approve

3. Building a Custom VPC with Terraform

A Virtual Private Cloud (VPC) is your isolated network in AWS. Terraform gives you granular control to define your VPC, subnets, route tables, and internet gateways, creating a secure and customized network environment.

Define VPC and Subnets

Start by defining the VPC's CIDR block, then create public and private subnets within it. 


# vpc.tf
resource "aws_vpc" "main_vpc" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_support   = true
  enable_dns_hostnames = true

  tags = {
    Name = "TerraformCustomVPC"
  }
}

resource "aws_subnet" "public_subnet" {
  vpc_id            = aws_vpc.main_vpc.id
  cidr_block        = "10.0.1.0/24"
  availability_zone = "${var.aws_region}a" # Example AZ
  map_public_ip_on_launch = true

  tags = {
    Name = "PublicSubnet"
  }
}

resource "aws_subnet" "private_subnet" {
  vpc_id            = aws_vpc.main_vpc.id
  cidr_block        = "10.0.2.0/24"
  availability_zone = "${var.aws_region}b" # Example AZ

  tags = {
    Name = "PrivateSubnet"
  }
}

output "vpc_id_output" {
  description = "ID of the created VPC"
  value       = aws_vpc.main_vpc.id
}

Internet Gateway & Route Tables

An Internet Gateway enables public internet access. Route tables dictate traffic flow within your VPC. 


# network_components.tf
resource "aws_internet_gateway" "main_igw" {
  vpc_id = aws_vpc.main_vpc.id

  tags = {
    Name = "MainInternetGateway"
  }
}

resource "aws_route_table" "public_route_table" {
  vpc_id = aws_vpc.main_vpc.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.main_igw.id
  }

  tags = {
    Name = "PublicRouteTable"
  }
}

resource "aws_route_table_association" "public_subnet_association" {
  subnet_id      = aws_subnet.public_subnet.id
  route_table_id = aws_route_table.public_route_table.id
}

resource "aws_route_table" "private_route_table" {
  vpc_id = aws_vpc.main_vpc.id

  tags = {
    Name = "PrivateRouteTable"
  }
  # Private subnets typically don't have a direct route to IGW
}

resource "aws_route_table_association" "private_subnet_association" {
  subnet_id      = aws_subnet.private_subnet.id
  route_table_id = aws_route_table.private_route_table.id
}
